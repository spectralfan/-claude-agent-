## 一、项目简介

**项目名称**：JChatMind — 多智能体协作 AI 编码平台

**一句话**：基于 Spring Boot 3.5 + Spring AI 1.1 构建的多智能体 AI 编码平台，支持 Scheduler-Worker-Reviewer 三层 DAG 编排、MCP 安全执行链与分层记忆系统，实现类 Claude Code 的自主开发能力。

**技术栈**：

| 层级 | 技术 |
|------|------|
| 后端 | Java 17, Spring Boot 3.5.8, Spring AI 1.1.0, MyBatis 3.0.3 |
| 数据库 | PostgreSQL (pgvector, JSONB) |
| 前端 | React 19, TypeScript, Vite 7, Ant Design X 2.0, Tailwind 4 |
| LLM | DeepSeek, 智谱 GLM-4 |
| 工具 | MCP (SSE 协议), Ollama (bge-m3 向量嵌入) |
| 消息 | SSE 实时推送, RocketMQ (可选) |

**核心能力标签**：
- 🤖 **多 Agent 协作**：Scheduler 拆解需求 → Worker 并行编码 → Reviewer 自动审查，全流程自动闭环
- 🔒 **安全工具执行**：三层 MCP 安全防线 + 结构化验证优先策略，杜绝 Agent 自主生成高危命令
- 🧠 **分层记忆系统**：WORKING/RECENT/ARCHIVE 三级记忆，结合向量检索与重要性评分
- ⚡ **实时可见性**：统一 SSE 协议覆盖 Agent 闭环与 Coding 全生命周期

## 二、简历正文（项目经历用）

### 项目简介

**JChatMind** — 基于 Spring Boot 3.5 + Spring AI 1.1 的多智能体协作 AI 编码平台，实现自主决策、多 Agent 协作编码、安全工具执行与分层记忆检索，支持 DeepSeek/智谱 GLM 多模型切换。

**技术栈**：Java 17 / Spring Boot 3.5 / Spring AI 1.1 / PostgreSQL pgvector / MyBatis / React 19 / Ant Design X / DeepSeek & GLM-4

---

### 技术亮点

1. **ReAct Agent 闭环引擎**：设计 PLANNING→THINKING→EXECUTING→OBSERVING 显式状态机，手动接管工具执行控制，按 tool 轮次智能裁剪上下文避免 LLM 400 错误。
2. **工具体系 + MCP 安全代理**：设计 20+ 本地工具灵活配置框架，集成 MCP 桥接与三层安全防护（结构化验证优先→策略门禁→参数改写），保障 Agent 工具调用安全可靠。
3. **Memory Hub 分层记忆**：设计 WORKING/RECENT/ARCHIVE 三级记忆，结合重要性评分与 Ollama bge-m3 + pgvector 向量检索，实现短期保持与长期语义检索。
4. **Session 文件持久化**：实现 JSONL/MD/JSON 三层文件持久化，管理会话生命周期状态机，乐观锁防并发冲突。
5. **SSE + EventBus 实时推送**：设计 12+ 事件类型的统一 SSE 协议，覆盖 Agent 闭环与 Coding 全链路，支持本地直推与 RocketMQ 生产部署切换。
6. **Coding 全栈 REST API**：统一 ApiResponse + 全局异常处理，提供 Agent/消息/知识库 CRUD、Coding 任务/工作区/Maven/Shell 执行等全链路端点。
7. **多 Agent DAG 编排**：设计 Scheduler/Worker/Reviewer 三角色编排引擎，基于 PostgreSQL 任务表驱动 DAG 状态机，按依赖解锁并行编码，终态自动续跑。

## 三、面试话术速查表

| 面试问题 | 回答要点 |
|---------|---------|
| 和普通 Agent 框架有什么区别？ | 手动 ReAct 闭环 + 多 Agent DAG 编排 + MCP 安全防线，不只是对话，而是自主编码平台 |
| 为什么不用 LangChain？ | Spring AI 与 Spring Boot 生态深度集成，自定义 ReAct 闭环更可控，避免框架黑盒 |
| DAG 调度遇到什么坑？ | 自动续跑可能形成无限循环，通过任务图终态检测 + max-round 限制解决 |
| ToolAware 消息窗口解决了什么？ | 深循环下普通滑动窗口裁断 tool 链，LLM 要求 tool 响应对齐 tool_calls |
| MCP 安全架构怎么设计的？ | 三层递进：结构化验证优先 → 策略门禁拦截 → 命令改写执行 |
| 分层记忆和简单 RAG 的区别？ | RAG 只做语义搜索；分层记忆加入重要性评分、三级自动流转、会话镜像 |
